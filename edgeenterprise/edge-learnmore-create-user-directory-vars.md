## <a name="path-variables"></a><span data-ttu-id="e8e24-106">路徑變數</span><span class="sxs-lookup"><span data-stu-id="e8e24-106">Path variables</span></span>

<span data-ttu-id="e8e24-107">用於修改資料目錄路徑的原則 (例如，設定 [UserDataDir](microsoft-edge-policies.md#userdatadir) 或 [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) 支援變數)。</span><span class="sxs-lookup"><span data-stu-id="e8e24-107">Policies for modifying data directory paths (For example, configuring the [UserDataDir](microsoft-edge-policies.md#userdatadir) or [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) support variables.</span></span> <span data-ttu-id="e8e24-108">設定這些原則時，可以使用變數來取代硬式編碼路徑。</span><span class="sxs-lookup"><span data-stu-id="e8e24-108">When configuring these policies, you can use variables instead of hard-coded paths.</span></span> <span data-ttu-id="e8e24-109">例如，將您的設定檔資料儲存在 Windows 的使用者本機應用程式資料下，而不是預設位置。</span><span class="sxs-lookup"><span data-stu-id="e8e24-109">For example, to store your profile data under user local application data on Windows instead of the default location.</span></span> <span data-ttu-id="e8e24-110">將 [UserDataDir](microsoft-edge-policies.md#userdatadir) 原則設定為 **${local_app_data}\Edge\Profile**。</span><span class="sxs-lookup"><span data-stu-id="e8e24-110">Set the [UserDataDir](microsoft-edge-policies.md#userdatadir) policy to **${local_app_data}\Edge\Profile**.</span></span> <span data-ttu-id="e8e24-111">在大多數 Windows 10 安裝中，此路徑解析為 *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*。</span><span class="sxs-lookup"><span data-stu-id="e8e24-111">On most Windows 10 installations, this path resolves to *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span></span>

>[!NOTE]
><span data-ttu-id="e8e24-112">若要查看目前**設定檔路徑**，請開啟**關於版本**頁面 (鍵入 "edge://version")。</span><span class="sxs-lookup"><span data-stu-id="e8e24-112">To view the current  **Profile path**, open the **About version** page (type "edge://version").</span></span> <span data-ttu-id="e8e24-113">**設定檔路徑**遵循以下格式：*C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\User Data\Default*。</span><span class="sxs-lookup"><span data-stu-id="e8e24-113">The **Profile path** follows this format: *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\User Data\Default*.</span></span>

### <a name="guidance-for-using-path-variables"></a><span data-ttu-id="e8e24-114">使用路徑變數的指南</span><span class="sxs-lookup"><span data-stu-id="e8e24-114">Guidance for using path variables</span></span>

<span data-ttu-id="e8e24-115">在使用路徑變數之前，請先查看以下指南。</span><span class="sxs-lookup"><span data-stu-id="e8e24-115">Review the following guidance before using variables for paths.</span></span>

- <span data-ttu-id="e8e24-116">涉及 Microsoft Edge 儲存不同資料之路徑的所有原則，都與平台相關。</span><span class="sxs-lookup"><span data-stu-id="e8e24-116">All policies that involve paths where Microsoft Edge stores different data are platform dependent.</span></span> <span data-ttu-id="e8e24-117">某些原則僅可用於特定平台，而其他原則可在所有平台上使用。</span><span class="sxs-lookup"><span data-stu-id="e8e24-117">Some of these policies are available only on specific platforms, but others can be used on all platforms.</span></span>
- <span data-ttu-id="e8e24-118">為了避免應用程式在不同場合從不同位置啟動而導致錯誤，請確保路徑是絕對的。</span><span class="sxs-lookup"><span data-stu-id="e8e24-118">To avoid errors caused by applications starting from different locations on different occasions, make sure that paths are absolute.</span></span>
- <span data-ttu-id="e8e24-119">每個變數只能在路徑中出現一次。</span><span class="sxs-lookup"><span data-stu-id="e8e24-119">Every variable can occur only once in a path.</span></span> <span data-ttu-id="e8e24-120">對於大多數變數來說，這是使用變數的唯一有意義方法，因為它們解析為絕對路徑。</span><span class="sxs-lookup"><span data-stu-id="e8e24-120">For most of them, this is the only meaningful way to use variables, because they resolve to absolute paths.</span></span>
- <span data-ttu-id="e8e24-121">如果路徑不存在，幾乎所有原則都會建立該路徑 (在現有情況下如果可能)。</span><span class="sxs-lookup"><span data-stu-id="e8e24-121">Almost all policies will create the path if it doesn't exist (if possible in the existing circumstances).</span></span>
- <span data-ttu-id="e8e24-122">由於 Microsoft Edge 不同版本/通道處理資料夾結構的方式不同，因此在某些原則中使用網路位置可能會導致意外的結果。</span><span class="sxs-lookup"><span data-stu-id="e8e24-122">Using network locations for some policies can lead to unexpected results due to differences in how different versions/channels of Microsoft Edge handle the folder structure.</span></span>
